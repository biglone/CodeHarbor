# Release v0.1.69 / 发布说明

## English

### Summary
- v0.1.69 hardens AutoDev validation gating with structured signals and reduces false negatives in completion-gate decisions.

### Highlights
- Added structured validation contract for executor output: `VALIDATION_STATUS` + `__EXIT_CODES__`.
- Updated completion-gate inference priority to rely on hard validation signals before text heuristics.
- Added regression tests for validation-status parsing, exit-code precedence, and contract prompts.
- Improved CI stability for queue transient-retry tests on slower runners.

### Suggested verification (optional)
1. Upgrade: `npm install -g codeharbor@0.1.69`
2. Check version: `codeharbor --version`
3. Run one AutoDev task and confirm completion-gate behavior with validation evidence.

## 中文

### 摘要
- v0.1.69 强化了 AutoDev 完成门禁的验证判定：优先使用结构化信号，减少误判。

### 重点更新
- 执行代理输出新增结构化验证契约：`VALIDATION_STATUS` + `__EXIT_CODES__`。
- completion gate 验证推断优先级升级：先看硬信号，再回退到文本启发式。
- 新增回归测试，覆盖验证状态解析、退出码优先级与契约提示词约束。
- 提升 CI 稳定性：放宽队列瞬时失败重试用例在慢速 runner 的超时预算。

1. 升级：`npm install -g codeharbor@0.1.69`
2. 检查版本：`codeharbor --version`
3. 跑一条 AutoDev 任务，观察 completion gate 是否按验证证据正确决策。
