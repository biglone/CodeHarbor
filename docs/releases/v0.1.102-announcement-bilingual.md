# Release v0.1.102 / 发布说明

## English

### Summary

- v0.1.102 delivers the T11 API lifecycle and multi-instance governance set in one converged release, with stronger diagnostics and CI coverage gates.

### Highlights

- API lifecycle is now complete for external task control:
  - `GET /api/tasks` with pagination and filters
  - `POST /api/tasks/:id/cancel`
  - `POST /api/tasks/:id/retry`
  - `GET /api/tasks/:id/events` (SSE stream)
- Multi-instance rate limiting now supports shared Redis mode with safe local fallback.
- Limiter observability now includes shared limiter status, deny reason split, rejection ratio, and recovery latency diagnostics.
- Added integration regression coverage for API/Webhook queue recovery after orchestrator restart.
- Added core-module coverage guard to CI/release workflow to prevent quality drift in critical modules.

### Suggested verification (optional)

1. Upgrade: `npm install -g codeharbor@0.1.102`
2. Check version: `codeharbor --version`
3. Exercise API lifecycle endpoints and verify SSE event streaming.
4. For multi-instance setups, verify shared limiter mode plus fallback behavior in diagnostics.

## 中文

### 摘要

- v0.1.102 以单版本收敛方式交付 T11 的 API 生命周期与多实例治理补全，同时补齐可观测性与关键覆盖率门槛。

### 重点更新

- API 生命周期能力补齐：
  - `GET /api/tasks`（分页与筛选）
  - `POST /api/tasks/:id/cancel`
  - `POST /api/tasks/:id/retry`
  - `GET /api/tasks/:id/events`（SSE 实时事件流）
- 多实例限流支持 shared Redis 模式，并具备本地回退机制。
- 限流观测能力增强：shared 模式状态、拒绝原因拆分、拒绝率与恢复耗时诊断。
- 新增 API/Webhook 队列在 orchestrator 重启后的恢复回归测试。
- 新增核心模块覆盖率守卫并接入 CI/Release 流程，防止关键模块质量回退。

### 推荐验证（可选）

1. 升级：`npm install -g codeharbor@0.1.102`
2. 检查版本：`codeharbor --version`
3. 验证 API 生命周期接口与 SSE 事件流行为。
4. 多实例场景下，验证 shared limiter 与本地回退诊断输出是否符合预期。
