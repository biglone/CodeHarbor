# Release v0.1.104

## Highlights

- Fixed Matrix session workspace recovery after service restart: resumed sessions now keep the original workdir instead of falling back to the room default.
- Persisted the effective chat workdir in SQLite session state so room/user conversations continue in the same repository or workspace across restarts.
- Added migration and restart regression coverage to lock the recovery path in both state-store and orchestrator tests.

## Upgrade

```bash
npm install -g codeharbor@0.1.104
```

## Verify

```bash
codeharbor --version
```

Expected output:

```text
0.1.104
```

## Notes

- This patch is recommended if you use long-lived Matrix room sessions that depend on a non-default workdir.
- After upgrade, restart the service and send a follow-up message in the same Matrix room/session to confirm the bot still operates in the original workspace.
